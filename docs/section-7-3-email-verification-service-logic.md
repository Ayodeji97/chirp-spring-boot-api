# Section 7.3 - Email Verification Service Logic

## What This Section Teaches

This section implements the **business logic** for email verification — creating verification tokens, validating them when the user clicks the link, and cleaning up expired tokens on a schedule.

## What Was Created

### `EmailVerificationService.kt` — Verification Logic

```kotlin
@Service
class EmailVerificationService(
    private val emailVerificationTokenRepository: EmailVerificationTokenRepository,
    private val userRepository: UserRepository,
    @param:Value("\${chirp.email.verification.expiry-hours}") private val expiryHours: Long
)
```

Three methods:

#### `createVerificationToken(email)` — Generate a New Token

```kotlin
@Transactional
fun createVerificationToken(email: String): EmailVerificationToken {
    val userEntity = userRepository.findByEmail(email)
        ?: throw UserNotFoundException()

    // Invalidate all existing unused tokens for this user
    val existingTokens = emailVerificationTokenRepository.findByUserAndUsedAtIsNull(user = userEntity)
    val now = Instant.now()
    val usedTokens = existingTokens.map { it.apply { this.usedAt = now } }
    emailVerificationTokenRepository.saveAll(usedTokens)

    // Create a fresh token
    val token = EmailVerificationTokenEntity(
        expiresAt = now.plus(expiryHours, ChronoUnit.HOURS),
        user = userEntity
    )

    return emailVerificationTokenRepository.save(token).toEmailVerificationToken()
}
```

**Flow:**
1. Find the user by email
2. Mark all existing unused tokens as used (prevents old tokens from working)
3. Create a new token with 24-hour expiry (auto-generated by `TokenGenerator`)
4. Save and return the domain model

**Why invalidate old tokens?** If a user requests verification multiple times, only the latest token should work. Old tokens are marked as used rather than deleted for audit trail purposes.

#### `verifyEmail(token)` — Validate and Verify

```kotlin
@Transactional
fun verifyEmail(token: String) {
    val verificationToken = emailVerificationTokenRepository.findByToken(token)
        ?: throw InvalidTokenException("Email verification token is invalid.")

    if (verificationToken.isUsed) {
        throw InvalidTokenException("Email verification token is already used.")
    }

    if (verificationToken.isExpired) {
        throw InvalidTokenException("Email verification token has already expired.")
    }

    // Mark token as used
    emailVerificationTokenRepository.save(
        verificationToken.apply { this.usedAt = Instant.now() }
    )

    // Mark user as verified
    userRepository.save(
        verificationToken.user.apply { this.hasVerifiedEmail = true }
    ).toUser()
}
```

**Flow:**
1. Look up the token in the database
2. Check it hasn't been used already (`isUsed`)
3. Check it hasn't expired (`isExpired`)
4. Mark the token as used (`usedAt = now`)
5. Set `hasVerifiedEmail = true` on the user

**Three distinct error messages** — each failure case gets a specific message so the user knows what went wrong (invalid, already used, or expired).

#### `cleanupExpiredTokens()` — Scheduled Cleanup

```kotlin
@Scheduled(cron = "0 0 3 * * *")
fun cleanupExpiredTokens() {
    emailVerificationTokenRepository.deleteByExpiresAtLessThan(now = Instant.now())
}
```

**Key concepts:**

- **`@Scheduled(cron = "0 0 3 * * *")`** — Runs daily at 3:00 AM. The cron format is: `second minute hour day-of-month month day-of-week`.

- Deletes all tokens where `expiresAt` is in the past. This keeps the database clean over time.

- Note: For `@Scheduled` to work, you'll need `@EnableScheduling` on a configuration class (comes in a later section).


Tokens expire after 24 hours. This is injected into the service via `@param:Value`.

## The `apply` Pattern

This section uses Kotlin's `apply` extensively:

```kotlin
verificationToken.apply {
    this.usedAt = Instant.now()
}
```

`apply` runs the block on the object and returns the object itself. It's a concise way to mutate JPA entities before saving — equivalent to:
```kotlin
verificationToken.usedAt = Instant.now()
verificationToken  // return the entity
```
