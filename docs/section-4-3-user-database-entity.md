# Section 4.3 - User Database Entity

## What This Section Teaches

This section bridges the **domain layer** and the **database** by creating a **JPA entity** (`UserEntity`) and a **Spring Data repository** (`UserRepository`). While the domain `User` model from Section 4.1 is a pure Kotlin data class, the `UserEntity` is annotated with JPA/Hibernate annotations so it can be mapped to an actual PostgreSQL table in Supabase.

## What Was Created

### `UserEntity.kt` вЂ” JPA Entity (Database Table Mapping)

```kotlin
package com.danzucker.chirp.infra.database.entities

import com.danzucker.chirp.domain.model.UserId
import jakarta.persistence.*
import org.hibernate.annotations.CreationTimestamp
import org.hibernate.annotations.UpdateTimestamp
import java.time.Instant

@Entity
@Table(
    name = "users",
    schema = "user_service",
    indexes = [
        Index(name = "idx_users_email", columnList = "email"),
        Index(name = "idx_users_username", columnList = "username"),
    ]
)
class UserEntity(
    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    var id: UserId? = null,
    @Column(nullable = false, unique = true)
    var email: String,
    @Column(nullable = false, unique = true)
    var username: String,
    @Column(nullable = false)
    var hashedPassword: String,
    @Column(nullable = false)
    var hasVerifiedEmail: Boolean = false,
    @CreationTimestamp
    var createdAt: Instant = Instant.now(),
    @UpdateTimestamp
    var updatedAt: Instant = Instant.now(),
)
```

**Key concepts:**

- **`@Entity`** вЂ” Marks this class as a JPA entity. Hibernate will manage its lifecycle and map it to a database table.

- **`@Table(name = "users", schema = "user_service")`** вЂ” Maps to a table called `users` inside the `user_service` schema. Using a dedicated schema keeps the user service's tables organized and separate from other services (like chat or notification).

- **`@Id` + `@GeneratedValue(strategy = GenerationType.UUID)`** вЂ” The primary key is a UUID, auto-generated by Hibernate. UUIDs are preferred over auto-increment integers because they're globally unique and don't leak information about how many users you have.

- **`@Column(nullable = false, unique = true)`** вЂ” Database-level constraints. `nullable = false` means the column cannot be NULL, and `unique = true` creates a unique constraint so no two users can have the same email or username.

- **`@CreationTimestamp` / `@UpdateTimestamp`** вЂ” Hibernate automatically sets `createdAt` when the entity is first persisted and updates `updatedAt` every time it's modified. These are audit fields вЂ” useful for debugging and analytics.

- **`var` instead of `val`** вЂ” Unlike the immutable domain model, JPA entities use `var` (mutable properties). Hibernate needs to be able to modify fields when loading data from the database or tracking changes.

- **`class` instead of `data class`** вЂ” JPA entities should not be `data class` because Hibernate uses proxies and lazy loading that don't work well with `data class` semantics (especially `equals`/`hashCode` based on all fields).

- **`@Index`** вЂ” Creates database indexes on `email` and `username` columns. Indexes speed up queries that filter or search by these columns (e.g., login by email, check if username is taken).

### `UserRepository.kt` вЂ” Spring Data JPA Repository

```kotlin
package com.danzucker.chirp.infra.database.repositories

import com.danzucker.chirp.domain.model.UserId
import com.danzucker.chirp.infra.database.entities.UserEntity
import org.springframework.data.jpa.repository.JpaRepository

interface UserRepository : JpaRepository<UserEntity, UserId> {
    fun findByEmail(email: String): UserEntity?
    fun findByEmailOrUsername(email: String, username: String): UserEntity?
}
```

**Key concepts:**

- **`JpaRepository<UserEntity, UserId>`** вЂ” By extending `JpaRepository`, Spring automatically provides implementations for common CRUD operations: `save()`, `findById()`, `findAll()`, `deleteById()`, etc. You don't write any SQL вЂ” Spring generates it at runtime.

- **Query method derivation** вЂ” Spring Data can generate SQL queries from method names:
  - `findByEmail(email)` в†’ `SELECT * FROM users WHERE email = ?`
  - `findByEmailOrUsername(email, username)` в†’ `SELECT * FROM users WHERE email = ? OR username = ?`

- **Nullable return types** вЂ” `UserEntity?` means the query might not find a match. This is safer than returning `null` without the type system knowing.

- **No `@Repository` annotation needed** вЂ” Spring auto-detects interfaces that extend `JpaRepository` and creates proxy implementations at startup.

## What Changed in `user/build.gradle.kts`

Added two dependencies:

```kotlin
dependencies {
    implementation(projects.common)

    implementation(libs.spring.boot.starter.data.jpa)  // NEW
    runtimeOnly(libs.postgresql)                         // NEW

    testImplementation(kotlin("test"))
}
```

- **`spring-boot-starter-data-jpa`** вЂ” Brings in Hibernate (JPA implementation), Spring Data JPA (repository abstractions), and HikariCP (connection pool). This is the core dependency for database access.

- **`postgresql`** вЂ” The PostgreSQL JDBC driver. Marked as `runtimeOnly` because your code never imports PostgreSQL classes directly вЂ” Hibernate uses it behind the scenes to talk to the database.

## Domain Model vs Database Entity

| | `User` (domain) | `UserEntity` (infra) |
|---|---|---|
| Purpose | Represents a user in business logic | Maps to the `users` database table |
| Annotations | None | `@Entity`, `@Table`, `@Column`, etc. |
| Mutability | `val` (immutable) | `var` (mutable, required by Hibernate) |
| Class type | `data class` | Regular `class` |
| Extra fields | None | `hashedPassword`, `createdAt`, `updatedAt` |

## How `ddl-auto: update` Works With This

When you run with the `dev` profile, Hibernate sees the `UserEntity` and:
1. Creates the `user_service` schema if it doesn't exist
2. Creates the `users` table with all the columns
3. Creates the indexes on `email` and `username`
4. On subsequent runs, it adds any new columns/indexes without dropping existing data
